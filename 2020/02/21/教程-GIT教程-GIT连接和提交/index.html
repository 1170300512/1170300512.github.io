GIT连接与提交

发表于 2020-02-21 更新于 2022-02-21 分类于 GIT笔记

简介

  git有四个工作区域，分别是：工作目录（Workspace）、暂存区（Index/Stage）、本地仓库（Repository）、远程仓库（Remote）

在已经下载GIT并安装和创建仓库的前提下

绑定GITHUB

  1. 查看本机是否安装 SSH,出现相关命令则安装成功

  2. 指定 RSA 算法生成密钥，然后敲四次回车键，之后就就会生成两个文件，分别为秘钥 id_rsa 和公钥 id_rsa.pub

  3. 文件的位置在 Git Bash 上面都有显示，默认生成在以下目录：

  1. 使用记事本打开id_rsa.pub并复制

  2. 进入我们的 GitHub 主页，先点击右上角，再点击 settings , 先点击 SSH and GPG keys，再点击 ***New SSH key.***新建SSH keys并填写标题,粘贴复制的内容

  3. 验证是否成功

GIT连接远程仓库
本地无文件

  1. 进入我们准备存储 Git 仓库的目录

  2. 将远程仓库 clone 到本地

  1. 设置用户名和邮箱
  1. 创建一个 text.txt 测试文件 , 查看仓库状态：
  1. 创建的文件 text.txt 没有被追踪，也就是没有提交到本地仓库。现在我们使用 git add 命令将文件添加到了[临时缓冲区] 。后面跟.是 会把本地所有untrack的文件都加入暂存区( 根据.gitignore做过滤 ),跟*是 把任何文件都加入 ,也可以直接跟文件名。再用 git commit -m “提交信息” 将其提交到本地仓库
  1. 查看仓库提交日志
  1. 将本地仓库提交到远程仓库，origin是远程主机的名字,master是远程分支名称， 第一次上传需要输入密码
本地有文件

  1. 使用 git init 命令初始化这个仓库 , 关联远程仓库 , 同步远程仓库和本地仓库
